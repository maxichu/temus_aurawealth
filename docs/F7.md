# F7 - Trace

## Feature Description

A basic trace logging system that records every user query submission.
Traces are stored in logs/traces.jsonl — one JSON object per line.
Each trace contains: timestamp, user, query, and workflow name.
The sidebar includes a "Trace Viewer" expander showing the latest 5 entries.

Implementation: lib/tracing.py — log_trace() writes to JSONL,
read_recent_traces() reads last N entries. App.py hooks log_trace()
into the chat input handler before the agent workflow runs.

---

## Test Case 1 — Single user trace creation

Scenario: Alice submits a question, and a trace entry is created in the log file.

 Steps:
 1. Select Alice in the sidebar
 2. Type "What is my largest holding?" and press Enter
 3. Open logs/traces.jsonl
 4. Inspect the file contents

Expected: One line is appended to traces.jsonl containing the correct
timestamp, user="Alice", query="What is my largest holding?", and
workflow="Planner->Research->Response".

Actual: PASS — log_trace() is called inside the chat_input handler with
st.session_state.current_user and prompt. The function opens the file
in append mode and writes the JSON line.

Status: PASS

## Test Case 2 — User identity tracking in traces

Scenario: Bob submits a question, and the trace log correctly records his
identity instead of the previously active user.

 Steps:
 1. Switch to Bob in the sidebar
 2. Type "How many positions do I own?" and press Enter
 3. Inspect the trace log for the last entry

Expected: The newest trace entry contains user="Bob", not "Alice" or any
other user.

Actual: PASS — log_trace() receives st.session_state.current_user as the
user parameter. When Bob is selected, the session state reflects Bob, and
the trace records his identity.

Status: PASS

---

## Test Case 3 — Multiple trace records in chronological order

Scenario: User submits 3 different questions. All 3 should appear in the
log file in the order they were submitted.

 Steps:
 1. Ask "What is an ETF?" as Alice
 2. Ask "How do bonds work?" as Alice
 3. Ask "What is dollar-cost averaging?" as Alice
 4. Open traces.jsonl and count the lines

Expected: Three JSON lines exist in traces.jsonl. Line order matches
question order. Each line has a unique timestamp.

Actual: PASS — each call to log_trace() appends one line to the file.
The file is opened in append mode (open with "a"), so lines appear in
chronological order.

Status: PASS
