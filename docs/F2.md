# F2 - Web Chat UI

## Feature Description

A browser-based chat interface built with Streamlit. Users type questions into a
chat input box and receive responses from the AI financial advisor.
Messages persist via session_state during the browser session.

Implementation: app.py — Streamlit app using st.chat_input, st.chat_message, and
st.session_state.messages for history.

---

## Test Case 1 — Sequential multi-question conversation

Scenario: User asks three consecutive financial questions in the same session.

 Steps:
 1. Open the app at http://localhost:8502
 2. Type "What is an ETF?" and press Enter
 3. Type "How do I choose between bonds and stocks?" and press Enter
 4. Type "What is a reasonable emergency fund size?" and press Enter

Expected: All three questions and their corresponding AI responses are visible
in the chat window as a continuous scrollable conversation.

Actual: PASS — each user message triggers the 3-agent workflow, and both the
user message and assistant reply are appended to st.session_state.messages.
The rendering loop displays all messages in order.

Status: PASS

---

## Test Case 2 — Page refresh clears chat history

Scenario: User has an active conversation, then refreshes the browser page.

 Steps:
 1. Type "What is a REIT?" and receive a response
 2. Refresh the browser page (F5 / Ctrl+R)
 3. Observe the chat window after reload

Expected: The chat window is empty after refresh. This is the intended behavior
(st.session_state is cleared on page reload).

Actual: PASS — st.session_state is browser-session-scoped. On refresh, the
session state is re-initialized, and "messages" is set to an empty list.

Status: PASS
