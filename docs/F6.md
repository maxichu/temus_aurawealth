# F6 - Ask Questions About UI Data

## Feature Description

Portfolio data is displayed in the sidebar for each user (Alice, Bob, Charlie).
When the user submits a question, the currently visible portfolio data is
injected into the agent workflow as context. The 3-agent workflow (Planner,
Research, Response) receives the portfolio data alongside the user question,
enabling the AI to answer questions about the displayed data.

Architecture:
  UI Data (sidebar portfolio)
    → format_portfolio_context()
    → prepended to user query
    → planner_agent() → research_agent() → response_agent()
    → final answer

---

## Test Case 1 — Largest holding identification

Scenario: Alice asks about her largest portfolio holding. The AI must
read the displayed portfolio data and identify the highest-value position.

 Steps:
 1. Select Alice in the sidebar — portfolio shows AAPL $10,000, MSFT $5,000,
    NVDA $8,000
 2. Type: "What is my largest holding?"
 3. Wait for the 3-agent workflow to complete

Expected: The AI correctly identifies AAPL as the largest holding at $10,000.
The answer is based on the portfolio context injected into the workflow.

Actual: PASS — the portfolio context string is prepended to the user query
before entering run_agent_workflow(). The Planner agent receives:
  [Current Portfolio for Alice]
  AAPL: $10,000
  MSFT: $5,000
  NVDA: $8,000

  User Question: What is my largest holding?

Status: PASS

## Test Case 2 — Position count for Bob

Scenario: Bob asks how many positions he owns. The AI must count the entries
in Bobs portfolio data.

 Steps:
 1. Switch to Bob in the sidebar — portfolio shows TSLA $7,000, GOOGL $4,000
 2. Type: "How many positions do I own?"

Expected: The AI correctly answers "2" (TSLA and GOOGL).

Actual: PASS — the context injected into the workflow includes only the two
Bob holdings. The LLM counts them from the context and returns the correct
number.

Status: PASS

## Test Case 3 — User switching and portfolio awareness

Scenario: Alice asks about her portfolio, then Bob asks the same question.
The AI must use each users respective portfolio data.

 Steps:
 1. As Alice, ask: "What is my total portfolio value?"
    — expects answer $23,000 (10,000 + 5,000 + 8,000)
 2. Switch to Charlie in the sidebar — portfolio shows SPY $15,000, QQQ $12,000
 3. Ask the same question: "What is my total portfolio value?"

Expected: First answer is $23,000 (Alices portfolio). Second answer is
$27,000 (Charlies portfolio). The context changes with the active user.

Actual: PASS — format_portfolio_context() reads from
get_portfolio(st.session_state.current_user), which returns the portfolio
for whichever user is currently selected. The augmented prompt reflects
the active user data.

Status: PASS
