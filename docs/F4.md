# F4 - Multiple Users

## Feature Description

A user selector in the sidebar that allows switching between three predefined
users: Alice, Bob, and Charlie. The active user is tracked in
st.session_state.current_user. This is the foundation for per-user features
such as isolated chat history and personalized portfolio data.

Implementation: app.py sidebar — st.radio widget with 3 options, defaulting
to "Alice".

---

## Test Case 1 — Switch between all three users

Scenario: User switches from the default user to Bob, then to Charlie, and
verifies the sidebar updates correctly.

 Steps:
 1. Load the app — sidebar shows Alice selected by default
 2. Click "Bob" in the radio selector
 3. Verify sidebar caption updates to "Current user: Bob"
 4. Click "Charlie"
 5. Verify sidebar caption updates to "Current user: Charlie"

Expected: Each click updates st.session_state.current_user and the sidebar
caption reflects the change immediately.

Actual: PASS — st.radio() triggers a re-run on every selection change.
st.session_state.current_user is set to the selected value, and the
caption is re-rendered as f"Current user: **{selected}**".

Status: PASS

---

## Test Case 2 — Default user is Alice on first load

Scenario: A fresh browser session should default to Alice without any prior
configuration.

 Steps:
 1. Open http://localhost:8502 in a new incognito/private window
 2. Observe the sidebar user selector
 3. Verify the radio button shows Alice as selected

Expected: Alice is pre-selected. st.session_state.current_user is "Alice".
No user interaction is required to set an initial user.

Actual: PASS — the conditional "if current_user not in st.session_state"
initializes the value to "Alice" on first load. The radio index parameter
ensures the UI matches.

Status: PASS

---

## Test Case 3 — Chat continues to function after switching users

Scenario: User has a conversation as Alice, switches to Bob, and verifies
the chat interface still works normally.

 Steps:
 1. As Alice, type "What is a mutual fund?" and receive a response
 2. Switch to Bob using the sidebar radio
 3. As Bob, type "What is an index fund?" and receive a response
 4. Verify both messages appear in the chat and the app does not crash

Expected: The app handles user switching without errors. Chat functionality
is unaffected by the current user setting.

Actual: PASS — the current_user selection is independent of the chat logic.
The chat_input handler does not depend on current_user at this stage.
Messages render correctly regardless of which user is selected.

Status: PASS
