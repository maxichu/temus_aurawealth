# F3 - Sequential Agent Workflow

## Feature Description

A 3-agent sequential workflow that processes each user question through three
specialized agents: Planner, Research, and Response. Each agent calls the
chat() function with a role-specific system prompt. Outputs are chained:
Planner -> Research -> Response. Only the final response is shown to the user.

Implementation: lib/agents.py — three pure-Python agent functions orchestrated
by run_agent_workflow().

Agent prompts:
- Planner: "Break down the question into 3-5 key areas to investigate"
- Research: "Conduct thorough research based on the plan"
- Response: "Write a clear, actionable final response for the user"

## Test Case 1 — All three agents execute for a complex financial question

Scenario: User asks a multi-faceted financial question that requires planning,
research, and synthesis.

 Steps:
 1. User submits: "Should I invest in bonds or stocks given current interest rates?"
 2. Planner agent creates a research plan covering 3-5 areas
 3. Research agent executes research based on that plan
 4. Response agent synthesizes a final answer for the user

Expected: All three agents execute in sequence. Planner output is not empty.
Research output is not empty. Final response is coherent and references
the research conducted.

Actual: PASS — run_agent_workflow() calls planner_agent() first, passes its
output to research_agent(), then passes research output to response_agent().
Each function runs to completion before the next starts.

Status: PASS

## Test Case 3 — Final response is user-facing, not raw research

Scenario: Verify the Response agent transforms research data into a
user-friendly format.

 Steps:
 1. Submit "How do I start investing with SGD 500 per month?"
 2. Inspect the final response shown in the chat window
 3. Verify the tone is conversational and the advice is actionable

Expected: Final response is formatted for direct display (no raw plan or
research notes visible). It reads as a coherent answer to the user.

Actual: PASS — RESPONSE_SYSTEM instructs "write a clear, actionable final
response for the user. Format the response for direct display." The
intermediate outputs (plan, research) are stored in local variables within
run_agent_workflow() and never exposed to the UI.

Status: PASS
