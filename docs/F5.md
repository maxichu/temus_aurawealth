# F5 - User-specific History

## Feature Description

Each user (Alice, Bob, Charlie) now has an independent chat history stored in
st.session_state.user_histories — a dict keyed by username. Switching users
loads that users conversation history into the chat view. New messages are
appended only to the active users history.

Implementation: app.py — replaced st.session_state.messages with
st.session_state.user_histories[username] for all read/write operations.

---

## Test Case 1 — Alice and Bob maintain independent conversations

Scenario: Alice sends a message, then Bob switches in and sees an empty chat.
Bob sends a message, then switching back to Alice shows her original message.

 Steps:
 1. Select Alice (default) in the sidebar
 2. Alice types "What are the best low-risk investments for retirement?"
 3. Verify the AI responds and the message appears for Alice
 4. Switch to Bob using the sidebar radio
 5. Verify Bob sees an empty chat window (no history)
 6. Bob types "How do I start investing with SGD 500 per month?"
 7. Verify Bob sees his own conversation
 8. Switch back to Alice
 9. Verify Alice still sees "What are the best low-risk investments for retirement?"

Expected: Alice and Bob histories are completely isolated. Switching users
does not leak messages between them.

Actual: PASS — st.session_state.user_histories is keyed by username. The
rendering loop reads exclusively from user_histories[current_user].
Append operations target the same key.

Status: PASS

## Test Case 2 — Multiple rapid switches maintain consistency

Scenario: User rapidly switches between all three users multiple times and
verifies history stays consistent.

 Steps:
 1. As Alice, type "Explain dollar-cost averaging." (message A1)
 2. Switch to Bob, type "What is a REIT?" (message B1)
 3. Switch to Charlie, type "How do bonds work?" (message C1)
 4. Switch back to Alice — verify only message A1 is visible
 5. Switch back to Bob — verify only message B1 is visible
 6. Switch back to Charlie — verify only message C1 is visible
 7. Repeat step 4-6 three times

Expected: Each users history is preserved across multiple switch cycles.
No messages are lost or mixed between users.

Actual: PASS — each users message list is an independent Python list object
in the user_histories dict. Switching reads the same list object via key
lookup, so it always retrieves the correct history.

Status: PASS

## Test Case 3 — Long conversation for one user does not affect another

Scenario: Alice has an extended conversation with multiple turns, while Bob
remains idle. Verify Bobs history is untouched.

 Steps:
 1. As Alice, ask 5 sequential questions:
    a. "What is an ETF?"
    b. "How do ETFs compare to mutual funds?"
    c. "What are the fees involved?"
    d. "Which ETF platform do you recommend?"
    e. "What is a good bond ETF?"
 2. Verify Alice sees all 5 questions and responses (10 total messages)
 3. Switch to Bob
 4. Verify Bob sees an empty chat window
 5. Bob asks one question: "What is a portfolio?"
 6. Switch back to Alice
 7. Verify Alice still sees all 5 of her questions and responses

Expected: Alice long conversation history (10 messages) remains intact.
Bob sees zero messages from Alice. Switching does not merge histories.

Actual: PASS — the user_histories dict stores each users messages as a
separate list. Alice 5-turn conversation creates 10 entries in
user_histories["Alice"]. Bob list (user_histories["Bob"]) remains empty
until he sends a message.

Status: PASS
